Module 01: Money, Ledger, Settlement

Collateralized credit = solvency constraints + liquidation throughput

flowchart TD
  A[Deposit collateral] --> B[Mint or borrow]
  B --> C[Collateral ratio monitor]
  C --> D{ratio >= threshold?}
  D -->|yes| E[Position remains open]
  D -->|no| F[Liquidation auction]
  F --> G[Debt repaid + collateral seized]
  G --> H[Protocol solvency restored]
  E --> C
  • Deposit 150 USDC, mint 100 debt units.
  • Collateral ratio = 150/100 = 150%.
  • Collateral falls to 120: ratio = 120%.
  • If liquidation threshold is 125%, position becomes liquidatable.
  • Oracle delay hides true ratio.
  • Keeper congestion delays liquidation.
  • Bad debt appears if collateral dumps faster than auctions clear.
