Module 00: Orientation - How to Think About DeFi

Content-rich explainer: protocol analysis lens before diving into specific markets

Why This Matters
  • Without a mechanism lens, protocol docs read like marketing.
  • The same analysis framework works across AMMs, perps, lending, and prediction markets.
  • You can detect hidden assumptions before production incidents do.

Every protocol can be reduced to state, transition function, invariants, and incentives. This is your universal x-ray. The point is not to memorize all designs; it is to quickly locate where value enters, where risk accumulates, and where assumptions can fail.

First Principles
  1. State is the source of truth (balances, debts, margins, liquidity positions).
  2. Actions mutate state under constraints.
  3. Invariants protect solvency and accounting consistency.
  4. Incentives determine whether participants maintain or exploit the system.
Worked Examples

Example 1: If a protocol offers 20% yield, trace exact payer path: fees, inflation, funding transfer, or hidden leverage.

Example 2: If liquidation depends on oracle updates every 60s, simulate a 30% move inside that window. What state remains stale?

Failure Modes
  • Invariants exist on paper but not enforced in all code paths.
  • Economic assumptions fail before smart contracts fail.
  • Protocol appears solvent until delayed accounting catches up.
What To Monitor
  • Distance to risk thresholds across positions.
  • Oracle freshness and update lag.
  • Execution congestion under stress.
Further Reading Prompts
  1. What is the minimal state set needed to simulate this protocol?
  2. Which actors are paid to keep the system healthy?
  3. Which assumptions are social/offchain rather than cryptographic?
