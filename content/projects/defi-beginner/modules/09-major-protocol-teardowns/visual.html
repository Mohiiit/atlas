Module 09: Major Protocol Teardowns

Applied mechanism comparison across Uniswap v4, perps, prediction markets, lending, and more

Why This Matters
  • This module converts isolated concepts into comparative protocol judgment.
  • You learn to evaluate designs by state transitions and risk ownership, not brand narratives.
  • Capstone outputs here become reusable due-diligence templates.
Capstone Flow
flowchart TD
A[Choose protocol set]-->B[Map state and transitions]
B-->C[Quantify PnL and fee paths]
C-->D[Stress failure modes]
D-->E[Compare trust assumptions]
E-->F[Write mechanism memo]
F-->G[Propose design improvements]

Use the same rubric for each teardown: state variables, invariant guards, payer/payee map, liquidation/recovery path, oracle/trust dependencies, and governance override powers.

Protocol Targets
  • AMMs: Uniswap v4, Curve, Balancer
  • Lending/stablecoins: Aave, Maker/Sky, Morpho
  • Perps: Hyperliquid, dYdX, GMX
  • Prediction markets: Polymarket, Augur-style
  • Shared security lens: EigenLayer and risk transfer
Failure Modes To Compare
  • Who absorbs tail losses first: user, LP, insurance, or tokenholders?
  • What happens when oracle or matching path degrades?
  • Can governance intervene quickly without creating capture risk?
What To Monitor
  • Realized execution quality vs stated model assumptions.
  • Incident response latency and transparency.
  • Parameter changes and measurable downstream effects.
