Unit 9

Oscillators, Phase Noise & Frequency Tracking

How the satellite's frequency reference works and why drift causes problems.

Learning Objectives

  • Explain the role of the oscillator as the frequency reference for all transmitted signals
  • Model frequency drift as a function of temperature using the polynomial drift equation
  • Interpret a phase noise spectrum and identify the key mask points in dBc/Hz
  • Derive how integrated phase noise translates into EVM and SNR penalty
  • Draw and explain the Phase-Locked Loop (PLL) block diagram and its lock conditions
  • Trace the thermal-electrical feedback loop from PA heating through oscillator drift to tracking degradation

9.1 What an Oscillator Does

Every radio system begins with a single, deceptively simple question: what frequency are we transmitting on? The answer comes from the oscillator -- a circuit that generates a periodic signal at a precise frequency. In Ku-band satellite communications, this means generating (or synthesizing from) a carrier near 12 GHz, which is 12 billion cycles per second.

The oscillator is the frequency reference for the entire transmit chain. The digital-to-analog converter (DAC) produces a baseband signal containing the modulated data (QPSK, 16QAM, etc.). The mixer multiplies this baseband signal by the oscillator's output to shift it up to the carrier frequency. Every modulation symbol -- every phase and amplitude point on the constellation diagram -- is defined relative to the oscillator's output.

Key insight: If the oscillator drifts by even a small fraction of its frequency, the entire transmitted signal shifts. The receiver must track this shift or it cannot demodulate the data. The oscillator is the metronome of the radio; if the metronome speeds up or slows down, the whole orchestra drifts.

In the simulator's RF chain, the oscillator sits between the DAC and the power amplifier. Its output feeds the mixer's local oscillator (LO) port. The quality of this signal -- its frequency accuracy and its spectral purity -- directly determines the quality of the transmitted signal.

9.2 Crystal Oscillators & TCXO

Satellites use crystal oscillators as their frequency reference. A quartz crystal is cut to vibrate at a precise frequency when an electric field is applied (the piezoelectric effect). The mechanical resonance of the crystal determines the output frequency, and quartz is chosen because its resonant frequency is extremely stable and predictable.

However, "extremely stable" is relative. A bare crystal oscillator's frequency varies with temperature, typically by several parts per million (ppm) per degree Celsius. For a 12 GHz carrier, 1 ppm equals 12 kHz -- a shift that is large enough to affect demodulation.

Temperature-Compensated Crystal Oscillator (TCXO)

A TCXO adds a temperature-compensation network around the bare crystal. A thermistor measures the crystal's temperature, and a correction voltage is applied to a varactor diode that fine-tunes the oscillation frequency. The goal: cancel the crystal's inherent temperature coefficient.

Typical TCXO stability is ~0.5 ppm/°C. This is 10-20x better than an uncompensated crystal, but still nonzero. Over the temperature swings a satellite experiences (the PA can heat the local environment by 30-50°C during a pass), even a TCXO accumulates meaningful drift.

On a mass-constrained LEO satellite carrying thousands of antenna elements, a TCXO is the pragmatic choice -- and the ground station's PLL compensates for its drift.

9.3 Frequency Drift Model

The simulator models frequency drift as a polynomial function of temperature deviation from a reference point (25°C). The model captures both the linear and quadratic components of crystal behavior:

Δf = fnominal × (α × ΔT + β × ΔT²) × 10-6

Where:

  • f_nominal = 12 GHz (the target carrier frequency)
  • ΔT = Tcurrent − Tref (temperature deviation from 25°C)
  • α = 0.5 ppm/°C (linear coefficient -- dominates for small ΔT)
  • β = 0.01 ppm/°C² (quadratic coefficient -- becomes significant for large swings)

Worked Example

Suppose the oscillator temperature rises from 25°C to 55°C (a ΔT of 30°C), which is realistic when the PA is operating at full power mid-pass:

Δf = 12 × 109 × (0.5 × 30 + 0.01 × 30²) × 10-6
Δf = 12 × 109 × (15 + 9) × 10-6
Δf = 12 × 109 × 24 × 10-6
Δf = 288 kHz

A 288 kHz offset at 12 GHz. The receiver's PLL must track this shift continuously or lose lock. Note how the quadratic term (9 kHz contribution from β) is already 37.5% of the linear term (15 kHz from α) at ΔT = 30. At larger swings, the quadratic dominates.

9.4 Phase Noise

Even if the oscillator's average frequency is perfect, its output is never a pure sine wave. Random fluctuations in the oscillator's phase -- caused by thermal noise in the crystal, amplifier noise, and power supply variations -- spread the oscillator's energy across a range of frequencies around the carrier. This is phase noise.

Phase noise is measured as L(f_offset) in dBc/Hz -- the noise power in a 1 Hz bandwidth at a given offset from the carrier, relative to the carrier power. Lower (more negative) values are better.

The Phase Noise Mask

A phase noise mask specifies the maximum allowed noise at specific offset frequencies. The simulator uses this mask for the satellite's oscillator:

Offset FrequencyPhase Noise (dBc/Hz)Meaning

1 MHz−130Far-out; mostly thermal floor

The phase noise rolls off roughly as 1/f² (20 dB/decade) for a well-designed oscillator -- this is the characteristic of white frequency noise passing through the oscillator's transfer function.

9.5 How Phase Noise Becomes EVM

Phase noise matters because it directly corrupts the modulated signal. Here is the mechanism, step by step:

  1. Phase uncertainty: The oscillator's phase jitters randomly. When the mixer upconverts the signal, this jitter is imprinted on every transmitted symbol.
  2. Symbol rotation: Each constellation point (QPSK, 16QAM, 64QAM) is defined by a precise phase and amplitude. Phase noise rotates each symbol by a random angle around its ideal position.
  3. Error vector: The distance between the ideal symbol position and the actual (rotated) position is the error vector. The RMS of all error vectors, normalized to the signal amplitude, is the Error Vector Magnitude (EVM).
  4. SNR penalty: EVM acts as an additional noise source. Higher EVM means the effective SNR is lower, regardless of how good the link budget is.

The Integration Formula

To convert phase noise into EVM, we integrate the phase noise power spectral density across the signal bandwidth:

σ²φ = 2 × ∫flowfhigh L(f) df    (integrated phase noise power, radians²)

EVMphase = σφ    (for small angles, sin(φ) ≈ φ)

EVMdB = 20 × log10(σφ)

The factor of 2 accounts for both sidebands. The integration limits are typically from 1 kHz (below which the PLL tracks out the noise) to half the symbol rate (the signal bandwidth). For a 240 MHz wide channel, fhigh = 120 MHz.

Worked Example

Using the phase noise mask as a worst case, and approximating the integral piecewise between mask points:

Segment 1 kHz--10 kHz: L ≈ −90 dBc/Hz (average), BW = 9 kHz
→ 10-9 × 9000 = 9 × 10-6

Segment 10 kHz--100 kHz: L ≈ −107.5 dBc/Hz, BW = 90 kHz
→ 1.78 × 10-11 × 90000 = 1.6 × 10-6

Segment 100 kHz--1 MHz: L ≈ −122.5 dBc/Hz, BW = 900 kHz
→ 5.6 × 10-13 × 900000 = 5 × 10-7

Total σ²φ ≈ 2 × 11.1 × 10-6 = 22.2 × 10-6 rad²
σφ ≈ 0.0047 rad ≈ 0.27°
EVMphase ≈ 0.47% (−46.5 dB)

This is comfortably below the EVM limits for QPSK (~17%) and 16QAM (~12.5%). But under stress -- when phase noise degrades by +10 or +20 dB due to PLL unlock or thermal effects -- EVM can spike to levels that force modulation downgrade.

Higher-order modulations are more sensitive. 64QAM requires EVM < 8%, meaning the allowed phase noise budget is ~4x tighter than for QPSK. This is why the simulator automatically drops to lower modulation orders when oscillator quality degrades.

9.6 Phase-Locked Loop (PLL)

The ground station cannot simply assume the satellite is transmitting at exactly 12 GHz. Due to oscillator drift, Doppler shift, and initial frequency uncertainty, the received signal could be offset by hundreds of kHz. The Phase-Locked Loop (PLL) is the circuit that tracks the satellite's actual frequency and keeps the receiver locked to it.

PLL Block Diagram

Figure 9.4 -- PLL block diagram. The phase detector compares the input signal to the VCO output; the loop filter determines tracking dynamics; the VCO generates the output frequency steered by the error signal.

How the PLL Works

  1. Phase Detector computes the phase error e(t) between the incoming signal and the VCO output. When locked, this error is near zero.
  2. Loop Filter smooths the error signal and determines the PLL's dynamic behavior. Its bandwidth BL determines how fast the PLL can track frequency changes. A typical value for satellite receivers is BL = 10-50 kHz.
  3. VCO adjusts its output frequency proportional to the control voltage v(t). When the error signal says "the input is higher than my output," the VCO speeds up.
  4. Feedback closes the loop: the VCO's output feeds back to the phase detector, so the loop continuously minimizes the phase error.

9.7 PLL Unlock

The PLL can only track frequency changes that fall within its loop bandwidth. When the frequency error exceeds the pull-in range (approximately ±2 × BL), the phase detector's output becomes incoherent -- the error signal wraps around, and the VCO cannot converge on the correct frequency.

This is PLL unlock -- a catastrophic (but temporary) event with several immediate consequences:

  • Phase noise spikes +20 dB: Without coherent tracking, the receiver's local oscillator is essentially free-running. All the noise that the PLL normally suppresses pours through.
  • EVM explodes: The phase uncertainty can reach tens of degrees. No modulation scheme above QPSK survives.
  • Reacquisition required: The receiver must perform a frequency sweep to find the signal again. This takes hundreds of milliseconds -- an eternity at 100+ Mbps where every millisecond represents ~12,500 packets.
Cascade effect: During PLL unlock, the simulator models the received signal quality as effectively zero. ModCod drops to the lowest rate (QPSK 1/4 or nothing), and the protocol layer reports massive packet loss. A 500 ms unlock event during peak throughput can cause a 60 Mbps goodput drop.

What Causes Unlock?

  • Rapid temperature change: If the PA suddenly increases power (or a thermal fault occurs), the oscillator temperature can jump faster than the PLL can track.
  • Doppler rate at low elevation: Near AOS/LOS, the satellite's radial velocity changes rapidly. Combined with oscillator drift, the total frequency change rate can exceed the PLL's slew rate.
  • Mechanical shock: Micrometeorite impact or thruster firing can cause crystal frequency jumps. These step changes are the hardest for a PLL to handle.
  • Power glitch: If the power bus voltage drops momentarily (eclipse transition, battery switchover), the oscillator frequency can jump.

In the simulator, PLL unlock is modeled as a discrete fault event with triggerTime, duration, and severity. Severity determines the phase noise penalty (+10 to +30 dB) and the reacquisition time.

9.8 The Thermal-Electrical Feedback Loop

This is where oscillator physics meets thermal physics in a closed loop that the simulator models tick by tick. The coupling chain:

  1. PA dissipates heat. The power amplifier converts DC power to RF power at 15-25% efficiency. The rest is heat. At 10 W RF output and 20% efficiency, the PA dissipates 40 W as heat.
  2. Oscillator warms. The PA and oscillator share the satellite's thermal bus. As the PA heats up, the oscillator's temperature rises -- not instantly, but with a thermal time constant of tens of seconds.
  3. Frequency drifts. The warmer oscillator shifts frequency per the drift model (αΔT + βΔT²).
  4. Tracking degrades. The ground station's PLL must track the changing frequency. If the drift rate approaches the loop bandwidth, tracking error increases.
  5. EVM rises. Increased tracking error plus elevated phase noise (phase noise worsens with temperature) both contribute to higher EVM.
  6. SNR penalty increases. Higher EVM means the effective SNR is lower than the link budget predicts.
  7. ModCod drops. The adaptive modulation system responds to lower SNR by selecting a more robust (but slower) modulation scheme.
  8. Goodput falls. Lower ModCod = fewer bits per symbol = lower throughput.
Figure 9.5 -- The thermal-electrical feedback loop. PA heat propagates through the oscillator to degrade tracking and throughput over ~30 seconds. Thermal inertia means recovery is slow even after the root cause is addressed.
The vicious cycle: In extreme cases, the PA's thermal throttling reduces output power to cool down, which reduces the link margin, which might cause the ground station to request lower ModCod, which in turn means longer transmission times for the same data -- keeping the PA hot longer. The simulator captures this interaction because the thermal model and the RF chain run at the same 1-second tick rate.

9.9 Code Connection

The oscillator model is implemented across two key files in the simulator:

src/simulation/rf-chain/oscillator.ts

This file contains the core oscillator model. The key function computes the frequency offset and phase noise for each simulation tick: 

// Frequency drift as function of temperature
function computeFrequencyOffset(
  fNominal: number,     // 12e9 Hz
  temperature: number,  // current oscillator temp (°C)
  tRef: number          // reference temp (25°C)
): number {
  const deltaT = temperature - tRef;
  const alpha = 0.5;   // ppm/°C linear
  const beta  = 0.01;  // ppm/°C² quadratic
  const ppmShift = alpha * deltaT + beta * deltaT * deltaT;
  return fNominal * ppmShift * 1e-6;
}

// Phase noise with temperature degradation
function getPhaseNoise(
  offsetHz: number,
  tempDegradation: number,  // dB penalty from temp
  pllLocked: boolean
): number {
  // Base phase noise from mask (interpolated)
  const basePn = interpolateMask(offsetHz);

  // Unlock adds +20 dB of phase noise
  const unlockPenalty = pllLocked ? 0 : 20;

  return basePn + tempDegradation + unlockPenalty;
}

src/simulation/rf-chain/signal-quality.ts

This file integrates the oscillator output into the overall signal quality calculation: 

// Convert integrated phase noise to EVM contribution
function phaseNoiseToEvm(
  integratedNoiseRad2: number
): number {
  // sigma_phi = sqrt(integrated noise power)
  const sigmaPhi = Math.sqrt(integratedNoiseRad2);

  // For small angles, EVM ≈ sigma_phi (radians)
  // Convert to percentage
  return sigmaPhi * 100;
}

// Total EVM from all sources
function computeTotalEvm(
  paEvm: number,        // from PA compression
  phaseEvm: number,     // from oscillator noise
  quantizationEvm: number
): number {
  // EVM contributions add in power (RSS)
  return Math.sqrt(
    paEvm ** 2 + phaseEvm ** 2 + quantizationEvm ** 2
  );
}
Root-sum-square (RSS): EVM contributions from different sources add in power, not linearly. This means the worst single contributor dominates. If PA EVM is 5% and phase EVM is 1%, total EVM is ~5.1% -- the phase noise barely matters. But if phase EVM jumps to 8% (PLL stress), total becomes ~9.4% and may exceed the modulation threshold.
