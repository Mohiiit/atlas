Unit 2

Phased Array Antennas

How Starlink steers its beam electronically, and how the simulator models element patterns, scan loss, and effective gain.

Learning Objectives

  • Explain why LEO satellites require phased arrays instead of mechanical dishes
  • Describe how phase shifts between elements steer a beam without moving parts
  • Define the element radiation pattern and interpret the cosq(θ) model
  • Calculate array gain from element count, element gain, and taper efficiency
  • Quantify scan loss from projected aperture and impedance mismatch
  • Predict beamwidth broadening with scan angle
  • Estimate the gain impact of element failures
  • Trace how the simulator codebase implements these antenna models

1Mechanical vs. Phased Array Antennas

A traditional parabolic dish antenna achieves high gain by focusing energy with a curved reflector. To track a moving satellite, the dish must physically rotate on a gimbal. This works fine for geostationary satellites (which appear fixed in the sky), but for LEO satellites like Starlink it creates two fatal problems:

  • On the satellite: A satellite serving thousands of users simultaneously would need to mechanically point different beams at different ground terminals. At 7.5 km/s orbital velocity, the geometry changes every second. Mechanical actuators cannot slew fast enough.
  • On the ground: The user terminal needs to track a satellite that crosses the sky in 10 minutes, then hand off to the next satellite. A motorized dish is expensive, bulky, and has moving parts that wear out.

Starlink solved this with flat-panel phased arrays on both the satellite and the user terminal. A phased array has no moving parts. It steers the beam electronically by adjusting the phase of the signal fed to each element, at microsecond timescales.

Figure 2.1 — A mechanical dish must physically rotate to change beam direction. A phased array adjusts element phases electronically and can steer in microseconds.

2How Phased Arrays Work

A phased array is a collection of antenna elements (typically microstrip patches at Ku-band) arranged in a regular grid. Each element radiates the same signal, but with a programmable phase delay. By controlling the phase difference between adjacent elements, the array creates constructive interference in a desired direction and destructive interference elsewhere.

The Physics of Beam Steering

Consider a linear array of elements spaced d apart. To steer the beam to angle θ from boresight (the direction perpendicular to the array surface), adjacent elements need a phase difference of:

Δφ = (2π / λ) × d × sin(θ)

where λ is the wavelength. When this phase progression is applied, the electromagnetic waves from all elements arrive in phase at angle θ, producing constructive interference (the main beam). At other angles the waves arrive out of phase and cancel, producing destructive interference (nulls and sidelobes).

Figure 2.2 — A linear array of elements with progressive phase shifts. The phase fronts are tilted, steering the main beam to angle θ from boresight.

Grating lobes: The element spacing d must be ≤ λ/2 to avoid grating lobes (spurious beams at other angles). For Ku-band at 12 GHz, λ = 25 mm, so λ/2 = 12.5 mm. This is the spacing used in the Starlink array model.

3Element Radiation Pattern

Each element in the array is a microstrip patch antenna. Its individual radiation pattern is well-approximated by a cosq(θ) envelope, where θ is the angle from boresight and q is a shaping exponent.

The cosq Model

Gelement(θ) = Gmax × cosq(θ)

In decibels:

Gelement,dBi(θ) = Gmax,dBi + q × 10 log10(cos θ)

What q Means Physically

  • q = 1: The radiation pattern equals the projected aperture factor. This is the theoretical minimum rolloff for a planar element.
  • q = 1.35 (used in the simulator): Accounts for additional rolloff from the patch's finite ground plane and substrate effects. This is a typical measured value for Ku-band patches with moderate substrate permittivity (εr ~ 3).
  • q = 2: A broader rolloff typical of elements with more directive substrate modes.

The simulator uses ELEMENT_MAX_GAIN_DBI = 5.0 and ELEMENT_PATTERN_EXPONENT = 1.35.

Figure 2.3 — Element radiation pattern. The solid curve shows cos1.35(θ) used in the simulator. The dashed curve shows cos(θ) for reference. Yellow dots mark the -3 dB points.

In the Codebase

From src/simulation/antenna/element-pattern.ts:

export function elementGain_dBi(theta_deg: number): number {
  if (absTheta >= 90) return -50; // back-hemisphere floor

  const theta_rad = (absTheta * Math.PI) / 180;
  const cosTheta = Math.cos(theta_rad);

  // G_element_dBi = G_max_dBi + q * 10*log10(cos(theta))
  return ELEMENT_MAX_GAIN_DBI
    + ELEMENT_PATTERN_EXPONENT * 10 * Math.log10(cosTheta);
}

4Array Factor & Total Array Gain

When N identical elements are coherently combined (all fed with the correct phase), the array gain is the product of the element gain and the array factor. In decibels:

Garray = 10 log10(N) + Gelement(θ) + 10 log10(ηtaper)

Breaking Down the Terms

TermValueExplanation
10 log10(N) 30.8 dB (for N=1200) The array factor. N elements coherently combined give N times the power of a single element. This is the fundamental advantage of a phased array.
Gelement 5.0 dBi at boresight Each individual patch element contributes its own gain (cosq pattern).
ηtaper 0.85 (-0.71 dB) Amplitude taper efficiency. Applying a Taylor window across elements reduces sidelobes but costs ~0.7 dB of directivity. The simulator uses TAPER_EFFICIENCY = 0.85.

Total at boresight: 30.8 + 5.0 - 0.7 = 35.1 dBi

Comparison with the FCC filing: SpaceX's FCC earth-station filing reports Starlink receive antenna gain of approximately 33.2 dBi. The simulator's 35.1 dBi at boresight is in reasonable agreement, with the difference accounted for by losses not included in the ideal array gain (ohmic losses, feed network losses, etc.).

Why 1200 Elements at λ/2 Spacing?

The Starlink user terminal uses approximately 1200 elements (this is a published estimate from teardown analyses). At λ/2 spacing for 12 GHz:

  • Wavelength: λ = c/f = 299,792,458 / 12×109 = 0.025 m = 25 mm
  • Element spacing: d = λ/2 = 12.5 mm
  • Array area: For a roughly square array of 1200 elements (~35 × 35), the aperture is about 35 × 12.5 mm = 437 mm ≈ 44 cm on a side

This matches the observed size of the Starlink Dishy McFlatface user terminal (a ~50 cm circular panel, with some dead area around the edges).

5Scan Loss

Scan loss is the single most important degradation in a phased array. As the beam steers away from boresight, gain drops due to three physical mechanisms:

Three Components of Scan Loss

ComponentCauseMagnitude at 60°
Projected aperture The effective area of the array as seen from angle θ is Aeff = A0 cos(θ). At 60°, the array presents half its area to the signal. -3.0 dB
Active reflection coefficient Scanning changes the mutual coupling between elements, shifting the active impedance away from the matched condition. Power is reflected back into the feed network. -0.5 to -1.5 dB (geometry-dependent)
Mutual coupling degradation Surface waves and higher-order modes between elements reduce radiation efficiency at scan. This is included in the impedance mismatch model. (included above)

Combined Scan Loss Formula

ηscan(θ) = cos(θ) × (1 - |Γactive(θ)|²)

where Γactive is the active reflection coefficient. In dB:

Scan Loss (dB) = 10 log10(ηscan)

Figure 2.4 — At boresight (left), the full aperture intercepts the signal. At 60° scan (right), the projected aperture is halved, contributing ~3 dB of loss. Impedance mismatch adds another ~1 dB.

Scan Loss Across the Pass

During a satellite pass, the steering angle equals approximately (90° − elevation). At low elevation (25° mask), the steering angle is ~65° with scan loss of ~4-5 dB. At TCA (82° elevation), the steering angle is only ~8° with negligible scan loss. This is a major reason why the link is weakest at AOS/LOS.

In the Codebase

From src/simulation/antenna/scan-loss.ts:

export function scanLoss_dB(theta_deg: number): number {
  // 1. Projected aperture loss
  const projectedAperture = Math.cos(theta_rad);

  // 2. Active reflection coefficient from impedance mismatch
  const gamma = activeReflectionCoefficient(clampedTheta);
  const mismatchFactor = 1 - gamma * gamma;

  // Combined scan efficiency
  const etaScan = projectedAperture * mismatchFactor;
  return 10 * Math.log10(etaScan);
}

The activeReflectionCoefficient() function models the impedance shift using a nearest-neighbor mutual coupling model with normalized self-impedance and coupling impedance terms.

6Beamwidth

The 3 dB beamwidth of the array broadens as the beam is steered off boresight. This is because the projected aperture shrinks, and a smaller effective aperture produces a wider beam (antenna reciprocity).

BW(θ) = BW0 / cos(θ)

where BW0 is the boresight beamwidth. For the ~1200-element Starlink array at Ku-band:

Steering Anglecos(θ)Beamwidth
0° (boresight)1.002.5°
30°0.872.9°
45°0.713.5°
60°0.505.0°
70°0.347.3°

Why This Matters: Pointing Loss

Wider beams are actually more forgiving of pointing errors. The pointing loss follows a Gaussian beam approximation:

Lpointing = 12 × (δ / BW)2  dB

where δ is the pointing error. With a typical 0.1° RMS jitter:

  • At boresight (BW = 2.5°): L = 12 × (0.1/2.5)² = 0.02 dB (negligible)
  • At 60° scan (BW = 5.0°): L = 12 × (0.1/5.0)² = 0.005 dB (even smaller)

Pointing loss becomes significant only during attitude disturbances or fault conditions, when jitter may spike to 0.5-1.0°.

7Element Failures & Graceful Degradation

A key advantage of phased arrays over dishes is graceful degradation. If a dish feed fails, the antenna is dead. If elements in a phased array fail, the remaining elements still form a beam, just with slightly less gain and slightly higher sidelobes.

Gain Impact of Element Loss

If Nactive elements remain out of Ntotal, the gain loss from the array factor alone is:

ΔG = 10 log10(Nactive / Ntotal)

For the simulator's four scenarios:

ScenarioElement DegradationActive ElementsArray Gain Loss
Clean0%12000 dB
Degraded5%1140-0.22 dB
Stressed12%1056-0.56 dB
Failing18%984-0.86 dB
Rule of thumb: Losing 10% of elements costs about 0.46 dB of gain. This is because 10 log10(0.9) = -0.46 dB. A phased array can tolerate significant element loss before the link budget is seriously affected. The sidelobe degradation (not modeled in detail here) is usually more concerning than the gain loss.

Beyond Simple Element Loss: Coherence

In reality, element "degradation" is not always binary (working vs. failed). Elements can have reduced gain, shifted phase, or intermittent behavior. The simulator models this through a subarray health field and computes a coherence loss that captures the penalty when elements are not perfectly phase-aligned:

function coherenceLoss_dB(subarrayHealth?: number[]): number {
  // Each subarray health value is 0..1 (0 = dead, 1 = perfect)
  const amplitudes = subarrayHealth.map(h => Math.sqrt(h));
  const ampSum = amplitudes.reduce((s, a) => s + a, 0);
  const powerSum = amplitudes.reduce((s, a) => s + a*a, 0);
  const efficiency = (ampSum * ampSum) / (N * powerSum);
  return -10 * Math.log10(efficiency);
}

This captures the fact that a non-uniform amplitude distribution across subarrays reduces the coherent combining gain below the ideal 10 log10(N).

The effective gain equation:

Geff = Garray + ScanLoss - PointingLoss - BeamShapeLoss - CoherenceLoss

This is called from the main simulation engine once per second. The steering angle comes from the pass geometry (90° − elevation), the active element count comes from the fault model, and the pointing error is generated deterministically from a seeded PRNG (Mulberry32).

  2. Calculate the total scan loss at a steering angle of 50°, considering only the projected aperture effect (ignore impedance mismatch). Then compute it including the impedance mismatch from the code's model (assume |Γ| ≈ 0.15 at 50°). How much does the impedance term contribute?
  3. A satellite pass has a maximum elevation of 40°. At AOS (elevation = 25°), the steering angle is 65°. At TCA (elevation = 40°), the steering angle is 50°. Calculate the difference in effective antenna gain between AOS and TCA, considering both the array factor (same) and scan loss (different). Assume no element failures.
  4. The simulator uses a Rayleigh-distributed pointing error with σ = 0.1°. Why Rayleigh and not Gaussian? (Hint: think about what quantity is being modeled.) What is the probability that the pointing error exceeds 0.3° (3σ)?
  5. Suppose 200 elements (out of 1200) fail completely. Calculate: (a) the gain loss from the reduced array factor, (b) the new boresight gain, and (c) qualitatively, what happens to the sidelobe level? Why might sidelobe degradation be a bigger concern than the gain loss for a satellite communications system?
  6. The activeReflectionCoefficient() function models impedance mismatch using nearest-neighbor coupling. At boresight, all elements have the same phase, so the mutual coupling is symmetric. At a scan angle, the phase progression breaks this symmetry. Explain intuitively why this causes the active impedance to shift away from the matched condition, and why the effect gets worse at larger scan angles.
