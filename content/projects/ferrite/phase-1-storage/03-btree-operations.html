B-Tree Operations

Three operations define a B-Tree: search, insert, and split. The first two are straightforward. The split is where the real engineering lives.

// Search returns a Cursor pointing to the key or its insertion slot
fn table_find(table: &mut Table, key: u32) -> Cursor {
    let root_page_num = table.root_page_num;
    let root_node = table.pager.get_page(root_page_num);
    if get_node_type(root_node) == NodeType::Leaf {
        leaf_node_find(table, root_page_num, key)
    } else {
        internal_node_find(table, root_page_num, key)
    }
}

Insert: Find Leaf, Shift, Write

Happy path (leaf not full)

  1. Run search to find insertion cursor
  2. Shift cells at and after cursor right by one slot
  3. Write key + serialized row at cursor position
  4. Increment num_cells
Cost: O(log n) to find the leaf, O(k) to shift at most k = MAX_CELLS cells. Since k is bounded by page capacity, it's effectively O(1) amortized.

Sad path (leaf is full)

  1. Run search to find insertion cursor
  2. Leaf is full — call leaf_node_split_and_insert
  3. Distribute N+1 cells across two nodes
  4. Update parent with new separator key + child pointer
Cost: O(log n) for the split propagation up the tree. Root splits are rarest — they increase tree height by exactly 1.

fn create_new_root(table: &mut Table, right_child_page_num: u32) {
    // The old root becomes the left child
    let left_child_page_num = table.pager.get_unused_page_num();
    // Copy old root bytes to left_child_page
    copy_page(table, table.root_page_num, left_child_page_num);

    // Re-initialize root page as an internal node
    let root = table.pager.get_page(table.root_page_num);
    initialize_internal_node(root);
    set_node_root(root, true);
    *internal_node_num_keys(root) = 1;
    *internal_node_right_child(root) = right_child_page_num;

    // Left child separator key = max key of left child
    let left_max = get_node_max_key(table.pager.get_page(left_child_page_num));
    *internal_node_key(root, 0) = left_max;
    *internal_node_child(root, 0) = left_child_page_num;

    // Update parent pointers on both children
    set_parent(table.pager.get_page(left_child_page_num),  table.root_page_num);
    set_parent(table.pager.get_page(right_child_page_num), table.root_page_num);
}
Root split is special: we can never move the root to a new page (page 0 is always root by convention). Instead, copy the old root contents to a fresh page (which becomes the left child), then reinitialize page 0 as the new internal root. Tree height increases by exactly 1.

// Distribute N+1 cells: right gets floor, left gets ceiling
const LEAF_NODE_RIGHT_SPLIT_COUNT: usize = (LEAF_NODE_MAX_CELLS + 1) / 2;    // 7
const LEAF_NODE_LEFT_SPLIT_COUNT:  usize = (LEAF_NODE_MAX_CELLS + 1) - LEAF_NODE_RIGHT_SPLIT_COUNT; // 7
