Node Layout on Disk

A B-Tree node is just bytes on a page. Understanding the exact byte layout is how you read and write nodes without an ORM — raw pointer arithmetic in Rust.

The Fundamental Unit: A Cell

A cell is the smallest addressable unit of storage in a leaf node. It pairs a key with its row data. Fixed-size cells make arithmetic simple: cell_offset = HEADER_SIZE + (cell_num * CELL_SIZE)

Leaf cell = 4 + 293 = 297 bytes
KEY
4 bytes (u32)
ROW DATA (serialized: id + username + email)
293 bytes
Why fixed-size? Variable-length cells require a free-space map per page. Fixed cells turn the page body into a simple array — cell N starts at a predictable offset. This is the tradeoff SQLite's toy tutorial makes; real SQLite uses variable-length records.

Rust Struct Definitions
// Common node header — sits at byte 0 of every page
// Offsets are computed as const usize values

const NODE_TYPE_OFFSET:   usize = 0;
const IS_ROOT_OFFSET:     usize = 1;
const PARENT_PTR_OFFSET:  usize = 2;
const COMMON_HEADER_SIZE: usize = 6;

// Leaf node header (follows common header)
const LEAF_NODE_NUM_CELLS_OFFSET: usize = COMMON_HEADER_SIZE;
const LEAF_NODE_HEADER_SIZE:     usize = COMMON_HEADER_SIZE + 4;

// Leaf cell layout
const LEAF_NODE_KEY_SIZE:   usize = 4;
const LEAF_NODE_VALUE_SIZE: usize = 293; // serialized Row
const LEAF_NODE_CELL_SIZE:  usize = LEAF_NODE_KEY_SIZE + LEAF_NODE_VALUE_SIZE;
const LEAF_NODE_MAX_CELLS:  usize = (PAGE_SIZE - LEAF_NODE_HEADER_SIZE) / LEAF_NODE_CELL_SIZE;

// Internal node header
const INTERNAL_NODE_NUM_KEYS_OFFSET:    usize = COMMON_HEADER_SIZE;
const INTERNAL_NODE_RIGHT_CHILD_OFFSET: usize = COMMON_HEADER_SIZE + 4;
const INTERNAL_NODE_HEADER_SIZE:        usize = COMMON_HEADER_SIZE + 8;

// Internal cell = child_ptr (4) + key (4)
const INTERNAL_NODE_KEY_SIZE:   usize = 4;
const INTERNAL_NODE_CHILD_SIZE: usize = 4;
const INTERNAL_NODE_CELL_SIZE:  usize = INTERNAL_NODE_CHILD_SIZE + INTERNAL_NODE_KEY_SIZE;

// Helper: pointer to key field of leaf cell N
fn leaf_node_key(page: &mut [u8], cell_num: usize) -> &mut [u8] {
    let offset = LEAF_NODE_HEADER_SIZE + cell_num * LEAF_NODE_CELL_SIZE;
    &mut page[offset..offset + LEAF_NODE_KEY_SIZE]
}
Why Fixed-Size Cells Win for a Toy DB

Zero fragmentation tracking

Cell N is always at HEADER + N*CELL_SIZE. No free list, no compaction, no slot directory needed.

Predictable split math

When full, split exactly at MAX_CELLS/2. No need to account for variable lengths when choosing the split point.
