Recursive Descent Parsing + AST

How a token stream becomes an Abstract Syntax Tree — one grammar rule, one function.

Parser Role

Tokens In, Tree Out

The lexer breaks text into tokens. The parser gives those tokens structure. It enforces grammar rules and organises tokens into a tree that reflects the logical meaning of the query — an Abstract Syntax Tree (AST).

A recursive descent parser implements the grammar directly: each grammar rule becomes one function. When a rule refers to another rule, that function calls the other function. The call stack mirrors the parse tree as it is built.

Grammar Rules

Simplified BNF for Our SQL Subset

These rules describe exactly which token sequences are valid. The parser enforces them by expecting specific tokens at each step.

statement = select_stmt | insert_stmt | create_stmt
select_stmt = SELECT columns FROM ident [ WHERE condition ]
insert_stmt = INSERT INTO ident VALUES ( value_list )
create_stmt = CREATE TABLE ident ( col_defs )
columns = * | ident ( , ident )*
condition = ident = value
value = integer | string_lit | ident

Square brackets [ ] denote optional elements. Parentheses group alternatives. The asterisk * after a group means zero or more repetitions.

Abstract Syntax Tree

The AST Rust Types

Each grammar rule maps to a Rust type. The top-level Statement enum covers all supported statement kinds:

enum Column {
    Wildcard,               // SELECT *
    Named(String),           // SELECT id, name
}

struct Condition {
    column: String,
    value:  Value,
}

enum Value {
    Int(i64), Str(String), Ident(String),
}

enum Statement {
    Select {
        columns: Vec<Column>,
        from:    String,
        where_:  Option<Condition>,
    },
    Insert {
        into:   String,
        values: Vec<Value>,
    },
    CreateTable {
        name:    String,
        columns: Vec<ColumnDef>,
    },
}

The root is the Statement::Select variant. Its fields — columns, the table name, and an optional WHERE condition — form the branches. Leaf nodes are concrete values: strings and integers.

Parse Trace

Following the Call Stack

Parsing SELECT id FROM users WHERE id = 1 triggers this function call sequence:

1 parse_statement() → sees Token::Select, calls parse_select()
2 parse_select() → expects SELECT, calls parse_columns()
3 parse_columns() → returns [Column::Named("id")]
2 parse_select() → expects FROM, reads "users"
3 parse_where() → sees WHERE, calls parse_condition()
4 parse_condition() → returns Condition { "id", Value::Int(1) }
1 parse_statement() → returns Statement::Select { columns, from, where_ }
Implementation

The parse_select Function

Each parser function consumes tokens it owns and delegates to sub-functions for nested rules. The expect helper asserts the next token matches and advances, or returns an error.

fn parse_select(&mut self) -> Result<Statement> {
    self.expect(Token::Select)?;        // consume SELECT
    let columns = self.parse_columns()?; // * or ident list
    self.expect(Token::From)?;          // consume FROM
    let from = self.expect_ident()?;    // table name

    let where_ = if self.peek() == Token::Where {
        self.advance();                  // consume WHERE
        Some(self.parse_condition()?)
    } else {
        None
    };

    Ok(Statement::Select { columns, from, where_ })
}

fn parse_columns(&mut self) -> Result<Vec<Column>> {
    if self.peek() == Token::Star {
        self.advance();
        return Ok(vec![Column::Wildcard]);
    }
    let mut cols = vec![Column::Named(self.expect_ident()?)];
    while self.peek() == Token::Comma {
        self.advance();  // consume comma
        cols.push(Column::Named(self.expect_ident()?));
    }
    Ok(cols)
}

Notice how parse_select calls parse_columns and parse_condition — functions that themselves consume tokens. This mutual recursion is the defining trait of recursive descent.
