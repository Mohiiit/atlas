Setup

Adding the Dependency

Add one line to Cargo.toml. The crate has no mandatory external dependencies and compiles quickly:

[dependencies]
sqlparser = "0.53"

Then import the dialect and parser in your source file:

use sqlparser::dialect::SQLiteDialect;
use sqlparser::parser::Parser;

The dialect controls which SQL extensions are accepted. SQLiteDialect permits SQLite-specific syntax like AUTOINCREMENT and double-quoted string literals.

Basic Usage

Parsing a SQL String

Parser::parse_sql returns a Result<Vec<Statement>> — a list because a single string can contain multiple semicolon-separated statements.

fn parse(sql: &str) -> Result<Vec<Statement>, ParserError> {
    Parser::parse_sql(&SQLiteDialect {}, sql)
}

fn main() {
    let sql = "SELECT id, name FROM users WHERE id = 1";
    let ast = Parser::parse_sql(&SQLiteDialect {}, sql)
        .expect("parse error");

    println!("{:#?}", ast[0]); // pretty-print the AST
}

The parsed AST for a SELECT statement comes back as a Statement::Query wrapping a Query struct, which contains the projection (columns), FROM clause, and an optional WHERE expression.

Pattern Matching

Extracting Data from the AST

Your executor pattern-matches on the top-level Statement enum to dispatch to the right handler. Unsupported variants return an error rather than silently ignoring input:

use sqlparser::ast::{Statement, SelectItem, TableFactor};

fn execute(stmt: &Statement) -> Result<(), String> {
    match stmt {
        Statement::Query(q) => {
            // Extract columns from the SELECT projection
            let body = &q.body;
            if let SetExpr::Select(sel) = body.as_ref() {
                for item in &sel.projection {
                    match item {
                        SelectItem::Wildcard(_) =>
                            println!("column: *"),
                        SelectItem::UnnamedExpr(e) =>
                            println!("column: {e}"),
                        _ => {}
                    }
                }
                // table name
                if let Some(from) = sel.from.first() {
                    if let TableFactor::Table { name, .. } = &from.relation {
                        println!("table: {name}");
                    }
                }
            }
            Ok(())
        }
        Statement::Insert { table_name, source, .. } => {
            println!("INSERT into {table_name}");
            // walk source rows ...
            Ok(())
        }
        Statement::CreateTable { name, columns, .. } => {
            println!("CREATE TABLE {name} ({} cols)", columns.len());
            Ok(())
        }
        _ => Err("unsupported statement".into()),
    }
}

The crate's AST is more verbose than a hand-written one — it covers every SQL construct. SetExpr::Select wraps the actual SELECT body because a query body can also be a UNION, INTERSECT, or VALUES clause.

Limitations & Dialects

What sqlparser-rs Does Not Do

⚠
Syntax only. The crate parses grammar but performs no semantic analysis. It will not tell you whether a referenced table exists or whether column types are compatible.
⚠
No execution. The AST is a data structure. Turning it into actual database operations is entirely your responsibility.
⚠
Dialect quirks. Some SQLite-specific constructs (like certain PRAGMA statements or ROWID) may require the GenericDialect as a fallback.
