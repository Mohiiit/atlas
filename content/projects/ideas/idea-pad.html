Kardomom
Active Fintech India
Browser-based credit card optimizer for the Indian market. At the point of purchase, it recommends the single best card from your portfolio — factoring in reward categories, cashback rates, and bonus multipliers. No real competitor currently exists in India. The insight is that most people leave significant rewards on the table by always using the same card.
Stack
PythonPlaywrightFastAPIBeanieMongoDBSolidJSTailwindCSS v4
Modules
Gecko (scraper) · Cardamom (API) · Chai (frontend)
Data source
PaisaBazaar — JS-rendered WordPress SPA, scraped via Playwright. 100+ cards across major Indian banks.
Auth
JWT + Google OAuth
Open problem
Normalizing reward points across banks (each bank has different point-to-INR ratios). Merchant category mapping for MVP is hardcoded — ML-based later.

Cloak
Active Starknet Agentic
Privacy-first smart wallet on Starknet. The goal is to become the unified privacy + agent payment layer for the ecosystem. Ships as a single SDK that developers npm install to get shielded payments, agent wallets, and micropayments in one package.
SDK
@cloak-wallet/sdk — TypeScript, published to npm. Mobile, web, and Chrome extension are reference implementations.
Privacy layer
Tongo ZK shielded pool — ElGamal commitments, nullifiers, ZK proofs. Payments are invisible on-chain (sender, amount, recipient hidden).
Payments
x402 micropayments (Coinbase standard, 500K+ txs week 1). Building z402-starknet — first shielded x402 implementation anywhere.
Key mgmt
Guardian-controlled delegation — 2-of-N multisig for recovery, ward model for scoped spending.
Current work
Shielded swaps via AVNU, agentic SDK integration, StarknetID / .stark names.

MEV on Polygon / Polymarket
Concept MEV
Polymarket is the leading prediction market, running on Polygon as a CLOB (Central Limit Order Book). The idea is to run a searcher bot that monitors the mempool and order book for extractable value — front-running large directional bets, arbitraging price discrepancies between Polymarket and Kalshi or Manifold, or exploiting resolution latency when real-world events settle.
Angles
Order-flow front-running · Cross-market arb (Polymarket vs Kalshi) · Resolution lag plays · Liquidity provision MEV
General MEV
Polygon PoS has validators (not miners), different MEV landscape. Flashbots has limited presence. Opportunity in DEX arb, sandwich, and liquidations on Polygon DeFi (QuickSwap, Aave).
Stack ideas
Rustethers-rsPolygon RPCwebsocket mempool
Status
Early-stage. No architecture yet — need to study Polymarket's smart contracts and CLOB mechanics first.

Indian Voice Developer Agent
Concept Voice AI India
A senior developer persona scoped to a codebase that you can have a real conversation with. Not a code search tool — an opinionated, senior-level AI that can discuss architecture decisions, debug problems, and push back on bad ideas. The key differentiator is the persona: speaks in a familiar Indian developer tone, knows the codebase deeply, and feels like talking to a teammate.
Input
Whisper (local STT) or cloud-based voice recognition with low latency
Indexing
tree-sitter for AST parsing, embeddings + vector DB (Qdrant/pgvector) for semantic search across the codebase
LLM
Claude as backbone — system prompt defines the senior dev persona, codebase context is injected via RAG
Output
TTS with Indian accent — ElevenLabs clone or fine-tuned Kokoro/XTTS model
Delivery
Could ship as a Claude Code hook, a daemon, or a VS Code extension

OpenCloaz as Google Home Device
Concept Local AI
Repurpose OpenCloaz (or a similar open-source voice assistant framework like OpenVoiceOS) as a local, always-on smart home device — a privacy-preserving alternative to Google Home. Everything runs on-device: wake word detection, STT, LLM inference, TTS. No cloud calls, no data leaving the box.
Hardware
Raspberry Pi 5 or a Mini PC (Beelink, N100). Needs a mic array and speaker.
Stack ideas
Whisper.cppOllamaPiper TTSopenWakeWord
Extensions
Home automation via Home Assistant. Could plug in the Indian Voice Dev Agent above for a coding-assistant mode.
Status
Pure concept. First step: get a local LLM talking via voice on a Pi.

Discord Voice Calls with Subagents
Concept Multi-Agent India
A team of AI agents that join a Discord voice channel — each with a distinct role (frontend dev, backend dev, PM, reviewer, QA). You can ask them questions, direct tasks, and have a real standup. Each agent has a different Indian accent and personality. Discord is just the interface; the underlying protocol could be extended to other platforms or a custom voice room.
Discord layer
discord.py + discord.js bots joining voice channels via Discord's voice gateway. Each agent = a separate bot with its own account and voice stream.
Voice
ElevenLabs voices cloned to Indian accents, or Coqui TTS with per-agent voice profiles. Each agent sounds distinct.
Brain
Claude multi-agent setup — each agent has a system prompt defining its role, domain knowledge, and personality. They coordinate via a shared context or message bus.
Use cases
Standups · Code review sessions · Architecture brainstorming · Rubber duck debugging with a whole team
