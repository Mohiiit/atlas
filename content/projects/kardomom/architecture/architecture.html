What is Kardomom? — Paste any Indian shopping URL (Amazon, Flipkart, Myntra, Swiggy, etc.) and Kardomom tells you which credit card from your wallet gives the highest rewards. It scores every card using a two-tier reward graph — direct cashback and hidden value from points-to-miles conversion. The URL trick: kardomom.com/https://amazon.in/dp/B0xxx routes directly to the optimizer.

Scraper Pipeline
Multi-Site BFS Crawler

Crawls Indian credit card aggregator sites using Playwright (headless Chromium). BFS discovery finds card URLs across bank listing pages, then extracts structured data with site-specific CSS selectors. Anthropic Claude LLM as fallback for complex fields.

Data Sources (Priority Order)
PaisaBazaar (P=4) CardInsider (P=3) BankBazaar (P=2) MyMoneyMantra (P=1)

PaisaBazaar is primary — richest data (reward categories, milestones, lounge access). Other sites fill gaps. Priority number drives merge order during consolidation.

Extraction Pipeline
  • Rule-based: per-site CSS selectors (e.g. PaisaBazaar: #benifits-section — note their typo)
  • LLM fallback: Anthropic Claude for fields CSS can't reach
  • BFS discovery: crawl bank listing pages, find card URLs automatically
  • Stealth: fake-useragent, random delays, playwright-stealth

Backend API
FastAPI Server

Async Python API with Beanie ODM. JWT auth (HS256, 7-day expiry) + Google OAuth via authlib. Pydantic response models for every endpoint. TDD approach with 10 test files.

  • POST /login — JWT token

  • GET /{slug} — full detail

/api/optimize
POST with URL or merchant name. Amount is auto-extracted from URL if not provided (via price_extractor). Defaults to ₹10,000 if extraction fails. Scores all cards (or portfolio only), returns ranked recommendations with first + second order breakdowns.

Price Extractor
URL → INR Transaction Amount

Fetches the product page HTML and extracts the price using a priority cascade of patterns. Amazon-specific patterns first (most reliable), then generic HTML patterns. Results cached in-memory for 10 minutes.

Amazon-Specific (High Confidence)
  • priceToPay.priceAmount — JSON (most reliable)
  • dealPrice.amount — sale/deal price
  • priceAmount — generic JSON field
  • ₹X with Y% savings — markdown format
  • M.R.P.: ₹X — fallback to MRP
  • Mirror mode: r.jina.ai bypass if bot-blocked
Generic HTML Patterns
  • <meta property="product:price:amount">
  • <meta itemprop="price">
  • "displayPrice": "₹X" — JSON in HTML
  • ₹X — generic rupee regex (lowest priority)

Cache: URL → (amount, source), TTL 600s. Amazon URLs normalized to /dp/{ASIN} for cache key.

Optimizer Scoring Engine
Two-Tier Reward Scoring

Each card is scored against the merchant's category using a reward graph stored in MongoDB. The optimizer computes two types of returns, then combines them with a discount factor:

1st Order Direct Spend Rewards

Immediate cashback or reward points earned on the transaction. Matches merchant category against card's spend_reward graph edges. Falls back to spend_reward_generic if no merchant-specific match.

points = (amount / 100) × best_points_per_100
value = points × point_value_rupees (default: ₹0.25/pt)

e.g. 5% cashback on ₹2,000 = ₹100

2nd Order Program Benefits

Hidden value from converting reward points into airline miles, hotel stays, or loyalty programs. Cards with transferable points can yield 2-5× more value. Uses program_benefit graph edges.

converted_value = points × transfer_ratio × mile_value
discounted at 60% (redemption uncertainty)

e.g. 4 pts/₹150 × 1:1 Airmiles × ₹1.2/mile
Final score = first_order_value + (0.6 × second_order_value).
Sanity guard: edges with >30% spend rate are rejected (malformed data).
Co-branded bonus: hardcoded lookup for card+domain combos (e.g. HDFC Amazon on amazon.in = 5%, Axis Flipkart on myntra.com = 4%)
Fallback Scoring (When Graph Unavailable)
Heuristic Pipeline
  • Check co-branded bonus first (3× weight)
  • Match reward_categories vs merchant category
  • Exact category match: 1.5× boost
  • Fall back to base_reward_rate
Category Multipliers
  • travel: 2.5×
  • online_shopping: 2.0×
  • food_delivery: 2.0×
  • grocery: 1.5× • fuel: 1.5×

Edge Fields
  • card_match_key — "issuer:name"
  • edge_type — spend_reward, spend_reward_generic, program_benefit
  • best_rate_percent
  • best_points_per_100
  • strength_score
Key Normalization
  • Issuer: "HDFC Bank" → "hdfc"
  • Name: strip issuer prefix, "Credit Card" suffix
  • Key: f"{issuer}:{name}"
  • Uses tuple-based pattern matching (not regex)
Edge Selection
  • Filter by edge_type + merchant
  • Sanity: reject >30% rates
  • Pick best by: (strength, rate, points)
  • Fallback: generic → heuristic
Frontend SPA
SolidJS Application

Reactive SPA built with SolidJS (not React). Signal-based state, no virtual DOM. <Show>, <For>, <Switch>/<Match> for rendering — never React patterns. TypeScript strict mode. TailwindCSS v4 with cardamom green palette + dark/light theme.

Architecture
  • AuthContext — JWT token + user signal
  • ThemeContext — dark/light toggle
  • client.ts — fetch wrapper (VITE_API_URL)
  • Vite proxy: /api → localhost:8000
  • Catch-all /*rest for URL passthrough
  • ?amount= query param support
Smart URL Routing — SmartRouter detects if the path segment is a valid URL (e.g. /https://amazon.in/dp/B0xxx) and routes to the Optimizer page automatically. Also parses ?amount= from query string to override auto-extraction.

Database Layer
MongoDB Collections
cards
CreditCard Beanie document. Indexes: slug (unique), issuer, created_at. Fields: name, slug, issuer, card_network, annual_fee, joining_fee, fee_waiver, reward_categories[], base_reward_rate, welcome_benefit, milestone_benefits[], lounge_access, fuel_surcharge_waiver, key_benefits[], tags[], image_url, source_url, needs_review, raw_text, last_scraped_at.
users
User Beanie document. Indexes: username (unique). Fields: email, hashed_password (argon2), google_id, portfolio[] (list of card slugs).
merchants
Merchant document. Unique index: domain. Fields: name, category, aliases[], logo_url.
scrape_runs
ScrapeRun audit trail. Fields: started_at, completed_at, urls_visited, cards_extracted, selector_failures[], errors[], seed_urls[].
reward_graph_materialized
Graph edges for optimizer. Keyed by card_match_key + graph_version. Edge types: spend_reward, spend_reward_generic, program_benefit. Fields: best_rate_percent, best_points_per_100, strength_score, merchant, program.
cards_consolidated + source collections
4 source collections (cards, cards_cardinsider, cards_bankbazaar, cards_mymoneymantra) feed into cards_consolidated during merge. Indexes: match_key (unique), issuer_key, name.
