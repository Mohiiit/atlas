What is Kardomom? — Paste any Indian shopping URL (Amazon, Flipkart, Myntra, etc.) and Kardomom tells you which credit card from your wallet gives the highest rewards. The URL trick: kardomom.com/https://amazon.in/dp/B0xxx routes directly to the optimizer.

Scraper Pipeline
Multi-Site BFS Crawler

Crawls Indian credit card aggregator sites using Playwright (headless Chromium). BFS discovery finds card URLs across bank listing pages, then extracts structured data with CSS selectors and LLM fallback.

Data Sources
PaisaBazaar CardInsider BankBazaar MyMoneyMantra
Extraction Strategy
Rule-based CSS selectors per site (primary) with Anthropic Claude LLM fallback for complex fields. Consolidation de-dupes and merges across all 4 sources.
CLI Commands
13 Typer commands: scrape batch discover bfs-crawl consolidate and more. Entry: uv run python -m kardomom_scraper
Output
CreditCard documents with: name, slug, issuer, fees, reward categories, lounge access, milestones, benefits, card images → MongoDB cards collection
Backend API
FastAPI Server

Async Python API with Beanie ODM, JWT authentication, and Google OAuth. Pydantic response models for every endpoint. TDD with 10 test files.

/api/auth
Register, login, me. JWT (HS256, 7-day expiry) + Google OAuth via authlib. Password hashing with argon2.
/api/cards
List cards (paginated, filterable by issuer/search). Get card by slug. Full detail with reward categories.
/api/portfolio
Get/update user's card portfolio. POST with card slugs. Drives personalized optimizer results.
/api/optimize
POST with merchant URL → scores all (or portfolio) cards. Returns ranked recommendations with first + second order returns.
/api/merchants
Resolve URL → merchant. Domain extraction, category mapping, price extraction from e-commerce HTML.

How the Optimizer Works
Scoring Engine

Each card is scored against the merchant's category. The optimizer computes two types of returns to find the best card:

1st Order Direct Rewards

The immediate cashback or reward points you earn on the transaction. Matches the merchant's category (e.g., "Online Shopping", "Travel", "Dining") against each card's reward categories.

reward_rate = card.category_match(merchant) × spend_amount
e.g. 5% cashback on "Online Shopping" × ₹2,000 = ₹100

2nd Order Points Conversion

The hidden value from converting reward points into airline miles, hotel stays, or other transfer partners. Cards with transferable points can yield 2-5x more value.

converted_value = points_earned × transfer_ratio × mile_value
e.g. 4 pts/₹150 × 1:1 Airmiles × ₹1.2/mile
Final score = max(first_order, second_order) − annual_fee_impact. Cards are ranked by effective return percentage. If the user has a portfolio, their owned cards are shown first with a separate "best overall" recommendation.
Frontend SPA
SolidJS Application

Reactive SPA built with SolidJS (not React!), TypeScript strict mode, Vite, and TailwindCSS v4. Signal-based state, no virtual DOM. Dark/light theme with cardamom green palette.

Database Layer
MongoDB Collections
cards
CreditCard document. Indexed by slug + issuer. Fields: name, fees, reward_categories[], lounge_access, milestones, benefits, tags, images.
users
User document. Indexed by username. Fields: email, hashed_password, google_id, portfolio[] (list of card slugs).
merchants
Merchant document. Unique index on domain. Fields: name, category, aliases[], logo_url.
scrape_runs
ScrapeRun audit trail. Fields: started_at, completed_at, urls_visited, cards_extracted, errors[].
