Biometric ZK Identity
POC Architecture

Three parallel approaches to proving phone biometric verification on Madara — from native signature verification to progressively more private ZK proofs.

A phone user authenticated via their device biometric (Face ID / fingerprint), and we can prove this on-chain — with varying degrees of privacy.

System Architecture
End-to-End Flow
From biometric touch to on-chain verified credential — every component in the pipeline.
Three Paths
Privacy Spectrum
Same biometric input, three different privacy guarantees. Each POC uses its own Madara devnet account.
A

Native secp256r1

Raw P-256 signature components (r, s) and public key (qx, qy) are submitted on-chain in plaintext. Cairo verifies the ECDSA math directly.

Hardware-only privacy No ZK ~27M L2 gas Low complexity
B

Noir + Garaga ZK

A ZK proof proves "I know a valid P-256 signature over this message" — the public key and signature are hidden inside the proof witness. This is the built POC-B, not v1 Option B.

Pubkey hidden in ZK UltraKeccakZKHonk ~200M Sierra gas Medium
C

Merkle + Nullifier

On-chain anonymity: proves set membership in a registered user group + nullifier prevents replay — two proofs from the same user cannot be linked by chain observers.

On-chain unlinkable ZK + Merkle set ~220M Sierra gas High
v1 Decision
On-Chain Claim Boundary
The built POCs prove raw biometric signatures. The preferred v1 interface can shift the trust boundary to a backend-signed credential bundle.
A

Raw P-256 in Circuit

The circuit verifies Apple's biometric P-256 signature directly. Strongest on-chain claim, highest proving cost. Current POC-B is closest to this.

Least backend trust Heaviest proof
B

Backend-Signed Bundle

The backend validates App Attest + P-256, then signs wallet + epoch. The circuit verifies the backend issuer signature. This is the preferred v1 direction.

Cheap circuit Trusted issuer key
C

Daily Fresh-Set Root

The backend publishes a signed Merkle root of fresh wallets for the day. Cheapest at scale, but adds batching and root-publication complexity.

Batchable Cheapest at scale
Step by Step
Complete Pipeline (POC-B as Reference)
The full journey of the current POC-B proof from biometric gesture to on-chain credential. Preferred v1 Option B would replace the raw P-256 proof with a backend-issuer-signature proof.
1

Biometric Capture

iOS Secure Enclave + CryptoKit
The app requests a one-time challenge from the relay. User authenticates with Face ID / Touch ID. The Secure Enclave uses a P-256 keypair with .biometryCurrentSet access control to sign the challenge. The private key never leaves the hardware.
2

Proof Package Assembly

BiometricProofService.swift
The app bundles: challenge ID, challenge hash, public key coords (qx, qy), DER signature parsed into (r, s), App Attest attestation/assertion data, and key backend identifier. This JSON package is POST'd to the relay at /demo/start.
3

Relay Validation

Python Flask — server.py + app_attest.py
The relay consumes the one-time challenge and performs two checks: (1) App Attest — validates Apple's certificate chain or refresh assertion, checks RP ID hash, and verifies the environment. (2) Signature — reconstructs the P-256 public key and verifies the ECDSA signature over the fresh challenge.
4

ZK Proof Generation

Noir + Barretenberg + Garaga
The current operator writes the raw P-256 witness to Prover.toml, compiles with nargo, generates a HONK proof, and converts it to Cairo calldata with garaga calldata. In preferred v1 Option B, this stage would prove a backend issuer signature instead.
5

On-Chain Submission

sncast invoke → Madara
The operator calls submit_biometric_proof(wallet, full_proof_with_hints). The protocol contract dispatches to the Garaga-generated Honk verifier. If verification passes, it extracts the message hash from public inputs and records expires_at[wallet] = now + 24h.
6

Credential Active

Cairo Storage — 24h TTL
Any dApp can now query is_human(wallet) which returns true if the credential hasn't expired. The contract emits a HumannessRefreshed event. The sample dApp reads this via Starknet JSON-RPC calls.
