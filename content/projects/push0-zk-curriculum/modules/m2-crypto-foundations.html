M2: Cryptography Foundations for ZK

DFS prerequisites: security games, adversarial model, commitments, hash functions, Merkle trees.

Core concepts

  • Completeness: honest prover convinces verifier.
  • Soundness: cheating prover cannot convince verifier except tiny probability.
  • Zero-knowledge: verifier learns nothing beyond statement truth.
  • Commitment schemes: bind now, reveal later.
  • Hash/Merkle commitments: compact authentication of large data.

Why push0 learners need this

push0 is orchestration, but the DAG tasks it schedules are cryptographic stages. Without these definitions, pipeline semantics lose meaning.

Exercises

  • Write a one-paragraph simulator intuition for zero-knowledge.
  • Map witness generation and commitment generation to security properties.
  • Identify where soundness can fail if prover stages are skipped or reordered.
