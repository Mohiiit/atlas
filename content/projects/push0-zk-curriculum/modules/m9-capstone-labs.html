M9: Capstone Labs (Build, Measure, Break, Fix)

Goal: prove you understand the stack by implementing and stress-testing a toy proving orchestrator.

Lab sequence

  1. Implement dispatcher + collector + durable queue interface.
  2. Add block ordering priority and retry with dead-letter queue.
  3. Add multi-input barrier collector for aggregation step.
  4. Inject failures: crash workers, delay tasks, duplicate deliveries.
  5. Measure P50/P95/P99 orchestration latency and queue growth.

Success criteria

  • No lost task across worker restarts.
  • No duplicate final aggregation outputs.
  • Oldest block priority maintained under load.
  • Clear latency budget decomposition.

Final oral exam prompt

"Why can a system with 5ms orchestration overhead still miss 10s P99 real-time targets, and how would you diagnose/fix it?"
